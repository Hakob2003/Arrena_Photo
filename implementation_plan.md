# Интеграция реальных платежей Stripe с Apple Pay и Google Pay

## Цель
Полностью заменить существующую заглушку оплаты на профессиональную, безопасную интеграцию Stripe с использованием `Stripe Payment Element`. Реализовать поддержку разовых покупок (кредитов) и подписок через кредитные/дебетовые карты, Google Pay и Apple Pay в тестовом режиме.

> [!IMPORTANT]
> **Требуется ваш ответ**
> - Нужно будет обновить схему базы данных (Prisma) для добавления `stripeCustomerId` и моделей для отслеживания истории платежей.
> - Есть ли у вас конкретные ID продуктов и цен в Stripe (Stripe Price IDs) для кредитов и тарифов, или мне стоит создать заглушки `price_xxx`, которые потом можно будет настроить через переменные окружения?

#### [MODIFY] `prisma/schema.prisma`
- Добавить `stripeCustomerId String?` в модель `User`.

## План тестирования и проверки

### Автоматические проверки
- Запустить `pnpm run lint` и `npx tsc --noEmit` по всему проекту.
- Проверить, что миграции Prisma применяются без ошибок (`npx prisma generate`).

### Ручная проверка
1. **Доступность Google Pay / Apple Pay**: Открыть модальное окно в Chrome (должен появиться Google Pay, если включен) и в Safari (Apple Pay).
2. **Покупка кредитов**: Выбрать пакет "500 Credits ($19.99)", ввести тестовую карту Stripe `4242...`, нажать Оплатить. Убедиться, что кредиты зачислились.
3. **Обновление подписки**: Выбрать "Pro Plan", провести оплату, проверить, что статус подписки изменился в БД через хук.
4. **Обработка ошибок**: Использовать специальные тестовые карты для отклонения (decline), чтобы убедиться, что пользователю выводится понятная ошибка без "падения" приложения.
