# 🛡️ QA Test Plan: Arrena Photo (Security & UI Release)

## 🎯 Цели тестирования

1. Убедиться в безопасности новой In-Memory JWT архитектуры (отсутствие XSS уязвимостей, успешное обновление сессии через Silent Refresh).
2. Проверить стабильность TypeScript-сборки (`next build`) и отсутствие регрессий.
3. Протестировать функциональность Истории платежей (формат времени, пагинация через URL, корректный Suspense-рендер).

---

## 🛑 Оценка рисков (Risk-based Testing)

- **Высокий риск (Критично):** `Auth Flow` — Пользователи могут быть "разлогинены" после обновления страницы, если Silent Refresh не сработает вовремя. Это заблокирует использование SaaS-платформы.
- **Средний риск:** Пагинация в биллинге. Использование `useSearchParams` может деоптимизировать роутинг, если `Suspense` был настроен неверно.
- **Низкий риск:** Отображение времени в секундах. Ошибки здесь повлияют только на UI.

---

## 🤖 Автоматизированные проверки (Must-pass checks)

- [x] **Lint & Types:** `npx tsc --noEmit` и `pnpm run lint` проходят без ошибок.
- [x] **Build:** `npx next build` завершен успешно. Нет ошибок, связанных с hydration или type mismatch.
- [ ] **E2E Smoke Path:**
  - Успешный вход в аккаунт (логин)
  - Обновление страницы (F5) на защищенном роуте — сессия должна сохраниться (Silent Refresh)
  - Выход из аккаунта (лог-аут)

---

## 🕵️ Ручное тестирование (Manual Checklist)

### 1. Авторизация и Сессии

- [x] Войти в систему, проверить отсутствие токена в `localStorage`.
- [x] Открыть консоль разработчика (Network) -> Обновить страницу -> Убедиться, что отправляется запрос `/auth/refresh` и возвращается статус 200/201.
- [x] Дождаться истечения `access_token` (или принудительно обновить его) — проверить, что фоновый refresh работает без прерывания пользовательского опыта.
- [x] Нажать "Выйти" — убедиться, что куки `refresh_token` удаляются, и `inMemoryToken` очищается. (Исправлен баг с отсутствием эндпоинта /auth/logout)

### 2. История Платежей (Billing / Payment History)

- [ ] Перейти на `http://localhost:3000/billing#payment`.
- [ ] Убедиться, что дата транзакций содержит время с секундами (например: `17.07.2026, 22:04:23`).
- [ ] Внизу таблицы выбрать страницу "2" — убедиться, что URL изменился на `?page=2` и страница не перезагрузилась полностью.
- [ ] Открыть в новой вкладке ссылку `http://localhost:3000/billing?page=2#payment` напрямую — убедиться, что открывается сразу нужная страница (Suspense не падает с ошибкой гидратации).

### 3. Edge Cases (Крайние случаи)

- [ ] Сброс кук вручную: удалить `refresh_token` в DevTools, обновить страницу -> пользователя должно перенаправить на `/login`.
- [ ] Ввод неверного `page` в URL (например, `?page=99999`) -> таблица должна показать пустое состояние ("No payment history found").
