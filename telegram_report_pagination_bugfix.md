# 🐛 Баг со сбросом фильтра устранен!

Извиняюсь за то, что проблема осталась в прошлый раз. Причина заключалась в особенности работы React Strict Mode и `useEffect`: при загрузке страницы скрипт автоматически перезаписывал сохраненное значение на стандартное (10) в момент загрузки.

## 🛠 Как было исправлено:
Теперь сохранение количества страниц в память браузера (`localStorage`) происходит **только в момент, когда вы физически выбираете новое значение в выпадающем списке**. Это на 100% исключает случайные перезаписи при навигации и загрузке.

**Ожидаемое поведение:**
1. Вы заходите в шаблоны. По умолчанию 10.
2. Вы меняете на 5. Страница сразу обновляется и показывает 5. В браузере запоминается 5.
3. Вы переходите в Профиль, затем возвращаетесь в Шаблоны.
4. При загрузке страницы шаблонов скрипт считывает память, видит 5, и загружает 5. Больше он ничего случайно не перезаписывает.

Ошибок TypeScript нет. Бэкапы обновлены, записи внесены.

Пожалуйста, проверьте еще раз! Теперь оно должно работать абсолютно железно.
